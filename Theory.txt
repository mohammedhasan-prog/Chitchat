Explained why a Python virtual environment (venv) is needed:
1. Dependency Isolation: Keeps project-specific dependencies separate, preventing version conflicts between different projects.
2. Preventing System Breakages: Avoids modifying the system-wide Python installation, which OS tools might rely on.
3. Reproducibility: Makes it easy to generate an exact `requirements.txt` to replicate the setup on other machines or for deployment.
4. No Admin Privileges Needed: Allows installing Python packages locally without needing administrator or root privileges.
